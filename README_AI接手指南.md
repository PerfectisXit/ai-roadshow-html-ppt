## 6. 当前格式与排版要求

这部分很重要。以后别再从旧文档拼规则，直接按这里。

### 6.1 全局视觉基调

- 整稿是暖白、偏产品化、偏展演现场的基调
- 不是深色科技风
- 不是纯静态海报风
- 不是传统 PPT 卡片堆叠风

默认优先级：

1. 讲演效果
2. 视觉秩序
3. 拟真组件
4. 文字说明

### 6.2 标题栏规则

- 所有正文页标题栏位置要统一
- 标题语法统一用当前 `html` 里的做法
- 前缀和主标题应在同一行
- 不要再把前缀单独做成一行小 kicker
- 正文页前缀默认沿用：
  - `板块一丨...`
  - `板块二丨案例一：...`
  - `板块三丨...`
  - `板块四丨...`
  - `板块五丨...`

如果未来要改标题：

- 优先克制、正经、有判断感
- 要像一页真正值得讲的问题
- 不要写得像口号
- 不要写得像 AI 给设计师的备注
- 不要太长

### 6.3 章节页规则

- 章节页的大标题与 `Chapter` 编号必须在同一行
- 章节页整体应水平、垂直都居中
- 章节页下面不要再挂解释性副标题
- 章节页标题尽量避免换行
- 横线长度跟随标题长度变化
- 各章节页位置、字号、排版语法要统一

### 6.4 封面页规则

- 当前封面是大气、居中、少卡片的处理方式
- 右侧不再挂辅助卡片
- 主标题、副标题、分享人、日期均为中心构图
- 分享人和日期在下方，但仍保持整体居中

### 6.5 页码规则

- 右上角页码位置固定
- 页码不能压到正文
- 页码区样式全稿统一
- 对观众可见页码只认 `p01 ~ p25`
- 章节页用 `c01 ~ c06`

### 6.6 字体与可读性

- 小字不能太小，必须考虑路演大厅投屏可读性
- 如果某页只能靠极小字号塞下内容，优先视为排版失败
- 优先处理方式：
  - 删字
  - 换行重组
  - 放大容器
  - 放大真正重要的字
- 不要默认靠缩小字号解决问题

### 6.7 留白处理

默认顺序不是“补文字”，而是：

1. 判断这页主事件够不够重
2. 判断主组件能不能更大
3. 判断组件内部能不能更饱满
4. 最后才考虑补新的辅助组件

不要做的事：

- 为了填空加说明卡
- 为了规整强拉卡片高度
- 用没有意义的装饰线、装饰进度条占地方
- 让一页出现大面积无意义空腔

### 6.8 文案红线

可见文案里不要出现：

- `接住`
- `不是……而是……`
- 设计者视角的提示语
- 给讲者自己看的台词说明
- 给 AI 自己看的注释型句子

文案默认要：

- 人话
- 克制
- 有判断
- 不油腻
- 不带明显 AI 味

## 7. 组件使用原则

新增或重做组件时，先问它属于哪一类：

1. 解释发生了什么
2. 证明为什么可信
3. 帮这一页快速收束

做不到这三点之一，就不要加。

当前项目的组件方向已经验证有效的是：

- 拟真聊天窗
- 案例工作台
- 桌面 Agent 任务流
- 终端安装/卸载流
- 时间轴与增长曲线
- 风险/偏差卡组
- 业务链路流程图
- 搜索/资料/文件上传下载类微组件

无效方向已经反复被证明包括：

- 纯说明卡堆字
- 左右各讲一遍同一件事
- 生硬复制同一种桥接动画
- 为了统一而让所有页长得一样

## 8. 流式输出组件硬规则

这是全稿已经反复踩坑后固定下来的硬规则，后续任何 AI 都必须遵守。

适用范围：

- 聊天窗
- 终端窗
- 日志窗
- 消息流
- 代码流
- 案例工作台左栏
- 桌面 Agent / 办公 Agent 的流式过程区

### 8.1 必须满足的 CSS 链路

- 整页包裹层：`height: 100%` + `min-height: 0`
- 主舞台容器：`height: 100%` + `min-height: 0`
- 内部滚动区：`overflow-y: auto` + `min-height: 0`
- 如果用了 `grid`，轨道优先用 `minmax(0, 1fr)`
- 如果用了 `flex`，子项显式写 `min-height: 0`
- 外层窗口通常还要 `overflow: hidden`

### 8.2 动画联动规则

- 不只是新消息插入时才滚到底
- 打字动画逐字出现时也要滚
- 插入代码块、文件卡片、预览块时也要滚
- 更稳妥的做法是：
  - 立即滚一次
  - 再在 `requestAnimationFrame` 里补滚一次

### 8.3 验收标准

只要出现以下任一情况，就算没做完：

- 内容把容器撑长
- 页面被顶出 16:9
- 底部黑边
- 底部越界
- 本应滚动的内容直接溢出到画幅外
- 打开后只剩黑屏或组件错位

一句话：

流式内容可以在窗里滚，但绝不能把外层舞台撑长。

## 13. 给未来 AI 的明确提醒

### 13.1 当前不是“搭框架阶段”

不要再默认：

- 先大改章节结构
- 先重做目录逻辑
- 先补新规划文件
- 先照着旧蓝图恢复以前删掉的页面

这些都不是当前默认任务。

### 13.2 当前最容易犯的错

- 看到旧文档就以为还在按旧分页施工
- 看到页面有空白就补说明卡
- 为了统一把所有页改成同一种结构
- 只看 DOM 不看截图
- 改流式组件时忘记固定高度链路
- 改完标题后没有检查全稿位置是否统一
- 用历史文件里的措辞把可见文案改回“AI 味”

### 13.3 如果用户让你继续改

默认先相信：

- 当前页码体系已经定了
- 当前章节顺序已经定了
- 当前大部分页面已经不是“待重构”，而是“待精修”

只有当用户明确要求推翻时，才进入结构级重做。
