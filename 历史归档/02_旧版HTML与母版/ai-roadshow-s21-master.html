用法：保留这份母版的标题区、页码、卡片语法和整体配色，只替换每页自己的内容与专属组件。 不要机械复制“聊天窗 + 右动作板”的长相，要复用它背后的秩序、对齐、密度与产品感。 有意义的页面前缀应尽量并入标题同一行，不要再单独显示成一行小字。 这份母版不是默认双栏模板；单栏、大字主句页、问题页、满屏主组件页都可以直接基于这套底板展开。
这份母版固定的是视觉底板，不固定结构比例。`master-stage` 可以改单栏，也可以换成 `ratio-equal / ratio-64-36 / ratio-72-28`。 遇到留白时，先判断空白区域更适合被主舞台长大吃掉，还是更适合补一个扫描、筛选、命中、追溯之类的动作型组件。 放大字号时要和外框、内边距一起协调放大；不同留白区的动效形式也不要同质化复用。 如果页面文字本来就不多，先减少文字、放大主句，再上一个更大的有意义动效组件，不要为了守双栏而补说明卡。 如果页面改完仍然发空，先反问：主组件是不是还没有真正贴着这页的讲演意图；其次再看组件内部和其他区块是否还可以一起做实。 任何时候都不要让文字量超过容器所能舒服承载的范围；一旦出现溢出、拥挤或硬塞感，优先删字、收句，必要时直接删掉那个组件。 卡片重叠、边框相压、模块遮挡是红线；一旦出现，就说明页面还没有收束好，不能交付。 处理留白时，不要把“单纯拉大空卡片”当成默认办法；应优先考虑更有表现力的动效组件、logo、图标、示意图或内嵌 SVG 图形。 讲到品牌、软件、平台，或“眼睛 / 手 / 大脑 / 扫描 / 命中 / 归并”这类抽象对象时，应优先考虑用标准 logo、图标或图形化表达去强化主题并消化留白。 新增组件必须“挣到自己的位置”：如果它既不帮助理解，也不帮助证明，更不帮助收束，就不要保留。 绝不允许把“先讲什么、后讲什么、这页作用是什么”这类注释语直接写进页面里；同一页也不允许为了规整而重复同一层信息。 真正开始改之前，要先过整章逻辑，再想清楚这页演讲者最想强调什么、观众最该先看到哪里，再决定页面结构与组件。
