## 第二阶段：核心组件开发（先造工具再改页面）

### H-01 增强版对话组件（Enhanced playChatDemo）
- **做什么**：重构 `playChatDemo` 函数，支持 8 种消息类型
  - text（现有）、file-upload、image-upload、file-output、image-output、code、chart、table
- **涉及代码区域**：
  - Block A（CSS）：新增 `.file-card`、`.file-output-card`、`.mini-bar-chart`、`.image-placeholder`、`.code-block-inline` 等样式
  - Block E（JS）：重构 `playChatDemo` 函数 + 新增 `renderFileUploadCard`、`renderFileOutputCard`、`renderChartShell`、`animateChartGrow`、`animateTableRows` 等辅助函数
- **产出**：增强版对话组件可用，但暂不改任何具体页面的 data-chat 数据
- **预估**：1-2 轮对话（这是最复杂的单个工作包）

### H-02 SVG Logo 内嵌系统
- **做什么**：将 `brandIcon` 对象中的外部 URL 替换为内联 SVG 字符串
- **涉及品牌**：OpenClaw、OpenAI/ChatGPT、Anthropic/Claude、DeepSeek、阿里云/QoderWork、Kimi、豆包、Midjourney、Sora
- **涉及代码区域**：Block B（`const brandIcon = {`）
- **附带改动**：全文搜索 `<img src="${brandIcon.xxx}"` 替换为直接输出 SVG 标签
- **产出**：所有品牌 logo 离线可用、代码内嵌
- **预估**：1 轮对话

### H-03 文件类型图标 SVG 组
- **做什么**：创建 PDF / DOCX / XLSX / PPTX 四种文件图标的 SVG 代码
- **用途**：供 H-01 的 file-upload 和 file-output 消息类型使用
- **产出**：4 个 SVG 模板函数，可根据文件类型参数返回对应图标
- **预估**：可与 H-01 合并在同一轮
