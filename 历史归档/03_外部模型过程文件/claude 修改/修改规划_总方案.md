# Part A：路演思路修改规划

## A-1 结构重组：从五板块到新五板块

### 当前结构的问题诊断

1. **板块四（怎么看待 AI）放在板块五（公司业务融合）之前，情绪节奏不对**。听众刚在板块三（工具地图）燃起来，板块四立刻泼冷水讲边界和红线，然后板块五又要把大家拉回来讲业务融合——像过山车，最后收尾缺乏升华感。
2. **板块一内部，从 OpenClaw 故事到"三次跳跃"认知框架的衔接不够有机**。"为什么偏偏在 2026 年"这个段落目前夹在故事和认知框架之间，像是一个补丁。
3. **板块三工具地图太平铺**。六个场景并列罗列，每个都讲一点，没有轻重缓急，观众记不住。桌面 Agent 作为你最希望大家关注的工具品类，在当前结构中淹没了。
4. **板块五结尾的"三阶段推进路径"过于自上而下**，不够接地气，员工听完可能觉得"又是一个方案"而不是"我明天可以做什么"。

### 新结构提议

```
板块一：一个故事——AI 已经"活"了
  ├─ 1A 封面/开场
  ├─ 1B OpenClaw 故事（含语音 → 六步链路 → 揭晓身份）
  ├─ 1C ★ Peter 进了 OpenAI → ChatGPT 5.4 发布 → 两者强强联合
  ├─ 1D ★ 三次跳跃认知框架（精简，作为"刚才的故事，其实就是这三跳的缩影"来引出）
  ├─ 1E Agent 与 Vibe Coding 两个钥匙概念
  │    → ★ 用 OpenClaw 故事回扣 Agent 定义："OpenClaw 就是 Agent 的活教材"
  │    → ★ 用 Peter 造 OpenClaw 的过程回扣 Vibe Coding
  └─ 1F 为什么 2026 年是特殊节点（窗口期收束）

板块二：三个案例——AI 怎么帮我们干活
  ├─ 2A 案例一：小王（第一跳能力演示）
  ├─ 2B 案例二：张姐（第二跳能力演示）
  └─ 2C 案例三：我自己 + Vibe Coding（第三跳能力演示）

板块三：怎么用好 AI——工具选择与桌面 Agent 深度体验
  ├─ 3A 工具地图（精简为一页总览 + 两个重点场景深讲）
  ├─ 3B ★★ 桌面 Agent 深度体验（Cowork / QoderWork 真实工作流演示）
  ├─ 3C 本地部署方案
  └─ 3D 提需求六步法

板块四（原板块五）：AI 与公司业务的深度融合
  ├─ 4A 导语：从"会用"到"深用"
  ├─ 4B ★ 新思路：不列三阶段规划，而是号召各部门梳理"可 AI 化的工作流"
  │    → 从最简单的 Vibe Coding 小工具开始
  │    → 再到固定化的 Agent 工作流
  │    → 逐步提高公司"AI 度"
  ├─ 4C 三个场景方向（数据平台 / 辅助决策 / 辅助运营）保留但精简
  └─ 4D 小结：由易到难的路径比三阶段规划更务实

板块五（原板块四 + 升华）：怎么看待 AI——边界、责任与未来
  ├─ 5A AI 的三种偏差
  ├─ 5B ★ 合规红线 + 工信部叫停 OpenClaw 截图案例
  ├─ 5C ★ 拔高：AGI 的概念与现阶段 AI 和 AGI 的差距
  ├─ 5D ★ 收束金句："与其等待 AGI 的到来，不如先用好手边的工具"
  └─ 5E 行动号召 + 结束页
```

### 调整理由逐条说明

#### 调整 1：板块四和板块五对调
- **核心逻辑**：先讲"公司怎么融合"（趁大家还有行动热情），最后讲"怎么看待 AI"（收束升华）。
- **情绪曲线**：板块三讲完工具，观众正好在"跃跃欲试"的峰值，此时讲公司业务融合是最好的承接；而"边界与红线"放到最后，配合 AGI 概念的升华，能让整场结束在"既有敬畏感又有行动力"的状态。

#### 调整 2：板块一内部重排——用 OpenClaw 回扣 Agent 概念
- **当前问题**：OpenClaw 故事讲完 → 三次跳跃 → 才到 Agent 定义，中间隔了太多认知框架，等讲到 Agent 时故事已经凉了。
- **新做法**：三次跳跃精简后紧跟故事，然后在讲 Agent 概念时直接说："刚才 OpenClaw 的六步链路，就是 Agent 的活教材——收到目标、自主规划、自己找工具、一步步推进到完成。"这样概念不再是抽象定义，而是有故事锚点的。
- **Vibe Coding 同理**：Peter 不写一行代码造出 OpenClaw，就是 Vibe Coding 最好的案例。

#### 调整 3：板块三精简工具地图 + 深讲桌面 Agent
- **当前问题**：六个场景并列，每个浅尝辄止。桌面 Agent 是你最希望推广的品类，但目前只有一页带一个通用的 desktop demo。
- **新做法**：
  - 工具地图缩减为 **一页总览**（六场景速查卡片），不再展开每个场景。
  - 挑两个"最能体现当今 AI 能力上限"的场景做重点展示（建议：**深度分析写作** 和 **桌面 Agent**）。
  - 桌面 Agent 做 **2-3 页深讲**，包含真实工作流的仿真动画演示（见 Part B 组件设计）。

#### 调整 4：公司业务融合板块——从"三阶段规划"改为"梳理可 AI 化工作流"
- **当前问题**：三阶段推进路径（1-3 月 / 3-6 月 / 6-12 月）是标准的管理咨询框架，对员工来说太遥远。
- **新做法**：
  - 号召各部门以部门为单位，梳理当下最常接触的**固定化工作流**（比如"每月经营分析"、"合同审查"、"会议纪要"等）。
  - 以这些工作流为需求和痛点，**先从最简单的开始**——用 Vibe Coding 做个小工具（比如一个自动格式化的模板），或者设计一条固定的 Agent 工作流。
  - **由易到难地逐步提高公司的"AI 度"**，而不是等一个"AI 数据平台"从天而降。
  - 三个场景方向（数据平台 / 辅助决策 / 辅助运营）保留作为**中长期方向参考**，但不再作为主体，变为收束性的"我们最终要走到哪里"。

#### 调整 5：最后一个板块加入 AGI 升华
- **新增 AGI 概念**：简单讲清"AGI = 在所有认知任务上达到或超过人类平均水平的 AI"，然后讲清楚"我们现在离 AGI 还有多远"——ChatGPT 5.4 得了 75 分，距离 100 分还有距离；AI 还会犯三种偏差；还有很多它做不到的事。
- **收束金句**："与其等待 AGI 的到来，不如先用好手边的工具。未来不是等来的，是一步步走过去的。"——既升华，又鼓励行动。
- **工信部叫停 OpenClaw 案例**：用真实截图/案例告诉大家 AI 的风险和使用边界。这比抽象地讲"四条红线"更有冲击力。

## A-2 逻辑衔接优化——关键转场设计

### 转场 1：OpenClaw 故事 → Peter 进 OpenAI → ChatGPT 5.4

**当前问题**：OpenClaw 揭晓后直接跳到 Peter 进了 OpenAI，衔接生硬。

**建议口播衔接词**：
> "这个程序后来变成了 OpenClaw，在 GitHub 上收获了 24.7 万颗星。但故事还没有结束——OpenAI 的 CEO Sam Altman 亲自把 Peter 挖进了公司。Peter 进去之后，OpenAI 几乎在同一时期，打出了另一张更大的牌……"

**设计要点**：用"故事还没结束"保持悬念，用"另一张更大的牌"制造期待感。

### 转场 2：三次跳跃 → Agent 和 Vibe Coding 概念

**当前问题**：三次跳跃讲完后，Agent 和 Vibe Coding 像是另起一段新内容。

**建议口播衔接词**：
> "刚才我们用三次跳跃，把 AI 这四年的进化讲清楚了。但有两个词，是理解后面所有案例的钥匙——Agent 和 Vibe Coding。其实你们刚才已经见过它们了：OpenClaw 那六步链路，就是 Agent 最好的活教材；Peter 不写一行代码就造出 OpenClaw，就是 Vibe Coding 最直观的证明。"

**设计要点**：不是"接下来我们讲两个概念"，而是"你们已经见过了，现在我把名字告诉你们"。

### 转场 3：2026 窗口期 → 板块二案例

**当前问题**："为什么 2026 年"这个段落目前像补丁。

**建议**：将"为什么 2026 年"从板块一中间移到 Agent/Vibe Coding 之后、板块二之前，作为**板块一的收束**：
> "所以，2026 年是一个特殊的窗口期。Agent 刚刚成熟，工具刚刚实用，普通人和大公司还站在同一条起跑线上。这就是我们今天这场分享的最重要的背景。接下来，我不讲概念了——我给你们看三个真实的案例，看看 AI 到底能帮我们干什么。"

### 转场 4：板块三工具 → 板块四公司融合

> "刚才给大家梳理了工具地图，也深入体验了桌面 Agent。但光会用工具还不够——关键是，这些工具能怎么跟我们真实的业务结合起来？接下来这个板块，我想跟大家认真谈一谈。"

### 转场 5：板块四公司融合 → 板块五边界与未来

> "前面讲了这么多 AI 能做什么、怎么跟业务结合，大家可能已经在想'我明天就去试'。这很好。但在冲出去之前，有几件事我们必须先冷静地想清楚。"

## A-3 重点内容增强建议

### 增强 1：OpenClaw 故事中更深度地引出 Agent 概念

**当前做法**：故事讲完后，Agent 概念在 1-D 节单独定义。
**建议做法**：在 s08（OpenClaw 揭晓页）之后，新增或改造一页，标题类似：

> **"OpenClaw 就是 Agent 的活教材——收到目标、自主规划、自己找工具、一步步推进到完成"**

这一页用一个**对比式动画组件**：左侧是"传统 AI（电话客服模式）"的交互流程，右侧是"Agent（驻场实习生模式）"的交互流程，两者同时播放，让观众直观看到区别。

### 增强 2：桌面 Agent 深度体验

**当前做法**：s37 一页带一个通用 desktop demo，四个操作节点快速闪过。
**建议做法**：

**第一页——核心变化概述**（保留但精简 s37）：
精简文字，保留引用金句"AI 终于从动嘴变成动手"，但主组件改为一个**仿真的 Claude Cowork 工作界面**——模拟用户给 Cowork 一个任务，Cowork 拆分步骤、逐步执行的全过程动画。

**第二页（新增）——真实工作场景演示**：
标题："给 Agent 一句话，看它怎么把活干完"
用一个完整的 **Cowork 仿真动画**演示一个贴近公司业务的场景：
> "帮我把下载文件夹里的发票照片整理成报销汇总表，按日期排序，金额汇总，输出 Excel。"

动画展示：Cowork 界面 → 读取文件夹 → 识别发票 → 提取数据 → 生成表格 → 输出文件

### 增强 3：合规红线加入真实案例

**当前做法**：四条红线是纯文字列表。
**建议做法**：
- 在四条红线之前或之后，新增一个**真实案例展示**：工信部紧急叫停使用 OpenClaw 的新闻/截图。
- 口播："大家看，OpenClaw 这么火，但工信部在 3 月就紧急发文叫停。为什么？因为 OpenClaw 的运行机制，意味着你电脑上的所有数据——包括你的文件、你的密码、你的操作记录——都可能被它读取和上传。对于我们央企来说，这是绝对不可以接受的。"
- **这比四条红线本身更有说服力**，因为它用真实事件告诉大家"这不是假想的风险"。

### 增强 4：AGI 概念与升华收束

**新增内容**：在板块五最后，加入：

1. **一页 AGI 概念**：
   - 简单定义：AGI = 在所有认知任务上达到或超过人类平均水平的 AI
   - 当前差距：用一个可视化的进度条或雷达图展示——"语言理解 90%、逻辑推理 75%、创造力 40%、常识判断 60%、自主行动 30%"（数字是示意，需要有比较直观的表达）
   - 核心信息：我们离 AGI 还有距离，但这个距离在以年为单位缩短

2. **收束金句**：
   > "与其等待 AGI 的到来，不如先用好手边的工具。未来不是等来的，是一步步走过去的。"

3. **行动号召**（比当前更具体）：
   > "从今天开始，每个部门梳理一条可以 AI 化的工作流。不需要一步到位，先把最简单的那件事交给 AI 试试。三个月后，我们再开一次交流会，看看每个部门都做了什么。"

### 增强 5：案例部分的对话演示要更丰富

**案例一（小王）的改进**：
- 目前对话只有文字输入/输出，建议增加：**用户上传文档的动作**（对话框中出现一个"📎 英文行业报告.pdf 20页"的文件气泡）、**AI 输出中体现返回文档**（"📄 已生成：《班子简报_初稿.docx》800字"的文件卡片）。

**案例二（张姐）的改进**：
- 除了当前的数据表格和图表，增加：**上传图片的动作**（"📷 合锦观澜_现场照片.jpg"）、**AI 生成图表在对话中流式渲染**、**AI 返回 PPT 文件**（"📊 月度经营分析.pptx 已生成，共15页"）。

**案例三（我自己）的改进**：
- 这个案例最适合做 Vibe Coding 演示。建议有一页展示我和 AI 对话中，**AI 在对话框中直接输出一段 HTML 代码**，然后旁边有一个"预览窗口"**实时渲染**这段代码的效果——就是你现在看到的这个幻灯片本身的某个组件。这是"元演示"（meta-demo），非常有冲击力。

# Part B：HTML Slide 修改规划

## B-1 总体修改策略

### 原则

1. **减文字、增组件**：每一页的文字量压缩 30-50%，腾出的空间用动画组件或图形化表达填充。
2. **案例演示全部组件化**：所有"讲案例"的页面，主舞台必须是一个拟真动画组件，文字说明退到辅助位。
3. **对话组件大升级**：当前 `playChatDemo` 只支持纯文字气泡，需要升级为支持**文件上传气泡、图片气泡、文件输出卡片、代码块、图表渲染**等富媒体类型。
4. **Logo 和图标全部内嵌代码**：涉及 OpenClaw、ChatGPT、Claude、DeepSeek、QoderWork 等产品的 logo，用 SVG 内嵌实现，不外挂图片。
5. **微动画提升质感**：所有静态卡片都应有入场动画；关键数据有计数器效果；流程步骤有逐步点亮效果。

### 页面增删预估

| 操作 | 页面 | 说明 |
|------|------|------|
| 合并 | s34 + s35 | 工具地图总览和场景 1-3 合并为一页六场景速查 |
| 合并 | s36 | 场景 4-6 合并入上述速查页 |
| 新增 | Agent 活教材页 | 在 s17 附近，用 OpenClaw 故事回扣 Agent 概念 |
| 新增 | 桌面 Agent 深度演示页 | 在 s37 之后，Cowork 真实工作流仿真 |
| 新增 | AGI 概念页 | 在原板块四之后 |
| 新增 | 工信部叫停案例页 | 在红线页附近 |
| 改造 | s50 | 从"12 个月推进路径"改为"部门工作流梳理"思路 |
| 总计 | 约 +2~3 页（合并抵消新增后） | |

## B-2 可复用组件设计（核心，6 个组件）

### 组件一：增强版对话仿真窗口（Enhanced Chat Demo）

#### 设计理念
当前 `playChatDemo` 只支持纯文本消息。升级后支持富媒体消息类型，让对话演示更接近真实的 ChatGPT / Claude 使用场景。这是全场复用频率最高的组件，案例一、二、三以及桌面 Agent 页面都会用到。

#### 消息类型扩展

在 `data-chat` 的 JSON 数据中，每条消息增加 `type` 字段：

```
消息类型定义：
{
  role: "user" | "ai" | "system",
  type: "text" | "file-upload" | "image-upload" | "file-output" | "image-output" | "code" | "chart" | "table",
  text: "消息文本内容",
  // 以下为各类型的附加字段：
  fileName: "英文行业报告.pdf",     // file-upload / file-output 时使用
  fileSize: "20页",                 // file-upload / file-output 时使用
  fileIcon: "pdf" | "docx" | "xlsx" | "pptx",  // 文件图标类型
  imageSrc: "data:image/...",       // image-upload / image-output 时使用（可选，也可用占位图）
  imageAlt: "合锦观澜现场照片",     // 图片描述文字
  code: "const data = ...",         // code 类型时的代码内容
  codeLang: "javascript",           // 代码语言标签
  chartData: {...},                 // chart 类型时的图表配置
  tableData: [...],                 // table 类型时的表格数据
}
```

#### 各消息类型的渲染规则

**`file-upload` 类型（用户上传文件）**：
- 渲染为用户气泡内嵌一个文件卡片
- 卡片样式：圆角矩形，左侧是文件类型图标（PDF 红色、DOCX 蓝色、XLSX 绿色、PPTX 橙色，用 SVG 绘制），右侧是文件名 + 文件大小/页数
- 入场动画：整个文件卡片从右侧滑入 + 轻微缩放

**`image-upload` 类型（用户发送图片）**：
- 渲染为用户气泡内嵌一个图片预览
- 图片区域：带圆角的灰色占位框（如果没有真实图片），内有一个相机图标 + 图片描述文字
- 如果提供了 `imageSrc`（base64 或 data URI），则显示真实图片缩略图
- 入场动画：图片框从右侧滑入

**`file-output` 类型（AI 返回文件）**：
- 渲染为 AI 气泡内嵌一个"已生成文件"卡片
- 卡片样式：左侧大号文件图标，中间文件名 + 描述，右侧一个绿色对勾 ✓
- 文件图标根据 `fileIcon` 字段选择对应的 SVG
- 入场动画：先显示"正在生成…"的加载态（进度条动画），然后切换到完成态

**`image-output` 类型（AI 生成图片）**：
- 渲染为 AI 气泡内嵌一个图片生成过程
- 先显示一个占位框 + "🎨 正在生成图片…" 的文字
- 然后图片从模糊到清晰渐现（CSS filter: blur 从 20px 渐变到 0px）
- 如果没有真实图片，用渐变色块 + 描述文字代替

**`code` 类型（AI 输出代码）**：
- 渲染为 AI 气泡内嵌一个代码块
- 代码块样式：深色背景（#1e1e1e），等宽字体，左上角有语言标签
- 代码逐行流式输出（每行一个 setTimeout，模拟编码过程）
- 这个就是当前 s06 终端窗口效果的对话框内嵌版

**`chart` 类型（AI 生成图表）**：
- 渲染为 AI 气泡内嵌一个动态图表
- 图表从零高度动画生长（柱状图柱子从底部长出，折线图从左到右绘制）
- 使用纯 CSS/SVG 实现，不依赖外部图表库

**`table` 类型（AI 输出表格）**：
- 渲染为 AI 气泡内嵌一个数据表格
- 表格逐行淡入显示
- 低于阈值的数据自动标红（模拟张姐案例中的"低于 80% 标红"）

## B-4 微动画与质感提升清单

### 全局微动画

| 微动画类型 | 适用范围 | 实现方式 |
|-----------|---------|---------|
| 卡片入场 | 所有 `.card`、`.proof-card` | `opacity: 0 → 1` + `translateY(12px) → 0`，stagger 延迟 80ms |
| 数据计数器 | 所有数字展示（24.7 万、75 分等） | `countUp` 函数，从 0 滚动到目标值，duration 1.5s |
| 标签淡入 | 所有 `.token-chip`、`.pill` | 从 `scale(0.8) opacity(0)` 到正常，stagger 60ms |
| 进度条生长 | 时间轴页、AGI 差距页 | `width: 0 → target%`，`transition: width 1.2s ease` |
| 脉冲呼吸 | 当前活跃步骤、当前高亮项 | `box-shadow` 脉冲动画，`animation: pulse 2s infinite` |
| 打字光标 | 所有流式文字输出 | 末尾闪烁的竖线 `|`，`animation: blink 0.8s infinite` |

### s17 右栏微动画增强（你提到满意的方向）

当前 s21 右栏的"第一步：整段翻译"和"第二步：定向筛选"已有一定动效。建议增强为：

- **翻译演示**：左侧英文逐行淡出，右侧中文逐行淡入，中间有一条横扫的"翻译光束"
- **筛选演示**：所有标签先全部出现，然后不相关的标签渐渐灰化缩小，相关的标签放大高亮、归拢到中心

### 图标质感提升

**s49 自媒体运营页的图标**：

当前使用 emoji（🔍🧾✍️🖼️🚀📈），建议替换为 SVG 图标：

```
每个运营环节的图标设计：
- 选题研究：放大镜 + 趋势线的组合图标
- 素材收集：文件夹 + 下载箭头

- 图片生成：画框 + 魔法棒
```

## B-6 风险提示

1. **文件体积**：当前 HTML 已近 2 万行，增加 6 个组件 + 新页面后可能超过 2.5 万行。建议在实施过程中对已合并/删除的页面及时清理代码。
2. **组件一的复杂度**：增强版对话窗口涉及 8 种消息类型，建议先实现最核心的 3 种（text、file-upload、file-output），再逐步补充其他类型。
3. **SVG Logo 的版权**：品牌 logo 即使用 SVG 手绘近似版，也存在品牌形象风险。建议对于内部分享会场景可以使用，但如果要对外传播需要谨慎。降级方案（首字母 Avatar）更安全。
4. **动画性能**：多个复杂动画同时播放可能在低配电脑上卡顿。建议每页同一时间只有一个主动画在播放，其他元素用简单的淡入代替。
