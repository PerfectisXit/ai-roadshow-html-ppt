## §6 关键技术细节（新会话必读）

### 如何添加新消息类型到对话

在 slide 的 `data-chat` JSON 中，每条消息加 `"type"` 字段即可。示例：
```json
{"role":"user", "type":"file-upload", "fileName":"月报.xlsx", "fileSize":"1 sheet", "fileIcon":"xlsx"}
{"role":"ai", "type":"file-output", "text":"已生成：", "fileName":"简报.docx", "fileSize":"800字", "fileIcon":"docx"}
{"role":"ai", "type":"chart", "text":"图表如下：", "chartData":{"labels":["A","B"], "values":[80,60], "colors":["good","warn"]}}
{"role":"ai", "type":"table", "text":"汇总：", "tableData":[["项目","完成率"],["锦方","108%"],["锦度","70% 🔴"]]}
```
详细参考 `H01_使用参考.md`。

### 如何注册新的动画组件

1. 写 `buildXxx(root)` 和 `async playXxx(root, token)` 两个函数
2. 在 `DEMO_PLAYERS` 数组中加一行：`[".xxx-demo", playXxx]`
3. 在 slide HTML 中用 `<div class="xxx-demo" data-xxx='...'></div>`

### 如何新增页面

在 `const slides = [...]` 数组的目标位置插入新对象：
```js
{
  id: "sXX",
  layout: "demo",  // 或 statement / strategy / decision / atlas / governance
  kicker: "",
  title: "...",
  subtitle: "",
  html: `...`
}
```

### 如何调整页面顺序

直接在 slides 数组中移动对象的位置。页码由 `assignPageMetadata()` 自动重算。

### brandIcon 使用方式

现在 brandIcon 的值是 SVG 字符串，在模板字面量中直接用 `${brandIcon.openai}` 输出。
**不要再用** `<img src="${brandIcon.xxx}">` 格式。

### 设计规范快速提醒

- 母版固定：标题区位置/字号、右上角页码、暖白配色/圆角/阴影
- 母版非固定：是否双栏、左右比例、组件类型——跟着内容走
- 优先做动作型组件，不要堆文字
- 新增组件必须有讲演意义（帮助理解/证明/收束）
- 不要为了填空而补内容

## §8 用户的核心偏好提醒

1. **案例要多且真实**：slide 展示要接地气，通过案例演示来呈现，而非文字讲解
2. **动画组件要"高度拟真"**：对话框中要能体现上传文档、发送图片、AI 返回文件/图表的动作
3. **图标要有质感**：用 SVG 代码实现，不用 emoji，不外挂图片
4. **讲演效果优先**：一切设计服务于讲演，不是静态排版
5. **公司业务融合板块是重点**：三个方向（数据平台/辅助决策/辅助运营）每个都要深入，体现可执行可落地
6. **语言质感要高**：不拘泥于原稿措辞，该改就改
7. **路演思路以 V5 为准**：V3/V4 已过时
