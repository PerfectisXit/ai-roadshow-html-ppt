# HTML 路演版 PPT 全局修改执行清单

> 临时文件。  
> 作用：给整套全局修改过程做章节级执行清单，避免后续修改时忘记规则或顺序。  
> 全部修改完成后删除。

## 使用原则

- 每开始一章前，先重读：
  - `README_AI接手指南.md`
  - `路演思路.md`
  - `ai_roadshow_修改规格说明.md`
  - `ai-roadshow-s21-master.html`
- 先做章节级判断，再做单页设计
- 先想讲演策略，再动排版和动效
- 页面中绝不允许出现注释型、备注型、解释设计思路型文字
- 不允许为了版面饱满而重复同一层内容
- 动效/动画/拟真组件优先级高于“补说明卡”
- 图标、logo、示意图、内嵌 SVG 组件优先级也要显著提高
- 默认每页改完做截图复核；如果当前轮次已和用户约定“最后统一验收”，可省去逐页截图，但不能省去最终整章或全稿视觉回看
- 每章改完必须做章节级回归
- 卡片重叠、文字压边、容器互相遮挡属于红线，出现即不得交付
- review、修样式、定位页面时，优先使用稳定 `slide id`，不要再人工猜“第 N 页大概对应哪个 sXX”

## 全局总目标

- 整体风格统一到母版系统
- 每页有明确视觉重心
- 文字不过度啰嗦，不用文字硬填空间
- 动效和组件服务讲演，不为炫而炫
- 分页、合并、拆分和顺序调整都服从讲演效果

## 每章开始前固定问题

- [ ] 这一章到底要完成什么讲演任务？
- [ ] 这一章哪些页应该合并？
- [ ] 这一章哪些页应该拆分？
- [ ] 这一章的高潮页、转折页、收束页分别是哪几页？
- [ ] 这一章哪些页当前最影响讲演效果？

## 每页开始前固定问题

- [ ] 这页最重要的一句话是什么？
- [ ] 观众最应该先看哪里？
- [ ] 哪些文字可以删？
- [ ] 哪些措辞可以更短、更准、更利于口播？
- [ ] 这页到底只需要保留几句话？能不能更少、更重？
- [ ] 页面里有没有任何只对讲者/设计者/AI有意义的注释型文字？如果有，必须删掉并只留在代码注释里
- [ ] 页面里有没有为了撑满版面而重复表达同一层意思？
- [ ] 留白更适合让主舞台长大，还是补动作型组件？
- [ ] 这页新增的组件或动画，究竟是在帮助理解、证明还是收束？
- [ ] 这页的主组件，是否真的贴着演讲者这一页最想强调的意图？如果只是规整、好看，但不够“对题”，就要重做
- [ ] 如果页面还显得空，是不是应该先把主组件内部做实，或者把其他关键区块增重，而不是立刻补说明卡
- [ ] 这页所有文字量是否和各自容器大小匹配？如果已经出现拥挤、溢出或明显硬塞，必须删字、收句或删组件
- [ ] 这页的每个组件，是否真的“挣到了自己的位置”？如果既不帮助理解、证明，也不帮助收束，就删掉
- [ ] 这页有没有本应一眼扫过的小标题或短句被迫换行？若有，先调布局，再缩字，最后再精简文案
- [ ] 这页是否应该加入 logo / 图标 / 图形化表达，来强化主题并消化留白？
- [ ] 如果讲的是品牌、软件、器官、工具、扫描、命中、归并等具体对象，是否已经有合适的视觉符号？
- [ ] 如果卡片内部用到 CSS grid，显式行数和实际子元素数量是否匹配？有没有因为隐式行把正文或主组件挤进错误轨道，制造大块空白或重叠？
- [ ] 当前修改如果属于“维护性整理”，是否只做了安全重构，没有顺手改页面内容表达？

## 内容少但必须单独成页时的固定处理顺序

- [ ] 先判断这一页是不是本来就应该“少而重”，不要先补文字
- [ ] 先放大真正重要的那一句话、那个判断或那个标题
- [ ] 再根据留白面积和形状，设计一个大小合适、并且有意义的动作型组件
- [ ] 同时优先考虑图标化、图形化、内嵌 SVG 组件，而不是只放大空卡片
- [ ] 不要默认做双栏；先判断单栏大字主句页、问题页、满屏主组件页会不会更好
- [ ] 组件必须服务表达：解释、可视化说明、流程示意、映射、扫描、命中、归并都可以
- [ ] 不允许通过增加备注型文字、重复性文字来凑满页面
- [ ] 当页面仍然发空时，先判断该页的“空”是主组件不够成立，还是组件内部不够饱满，还是其他区块太轻
- [ ] 不允许只通过拉大卡片尺寸来“装满页面”，除非内部内容也同步被做实

## 统一验收标准

- [ ] 不为设计而设计
- [ ] 不为饱满而硬塞内容
- [ ] 一切为讲演效果服务
- [ ] 一切为把观众注意力带到演讲者想强调的地方服务
- [ ] 每页必须有明确视觉重心
- [ ] 每页文字必须有讲演作用
- [ ] 每页动效必须有表达作用
- [ ] 每页图标 / 图片 / 图形元素也必须有表达作用
- [ ] 页面结构必须服从内容，不得为了守双栏而硬补文字
- [ ] 页面中不能出现任何“给设计者/给讲者自己看的话”
- [ ] 同一页内不能有为了饱满而产生的重复信息
- [ ] 页面里的主组件是否足够贴近讲演者真正想让观众看到的现实场景或判断，而不是只显得规整
- [ ] 页面中绝不能出现卡片重叠、模块遮挡、文字压线等低级错误
- [ ] 给用户看的版本应尽量是已经收束、收敛、完成度高的版本，而不是明显半成品
